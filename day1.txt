서버기반VCS (remote storage 서버저장소)
각자의 컴퓨터의 워킹코피가 최신업데이트된 레퍼지토리(서버 저장소)의 내용을 떙겨온다(업데이트). 그 후 워킹코피에서 작업한 내용을 래파지토리에 커밋
작업한 내용을 저장할 별도의 저장소가 없고
서버 저장소 하나만 있음
서버에 접속할 수 없거나 인터넷이 안되면 작업 불가능

분산형VCS remote storage > local
서버 저장소의 하위에 개별 저장소(로컬저장소)를 가짐
각 컴퓨터의 위킹코피는 개별 저장소를 가지고 있음
워킹코피의 소스를 로컬저장소에 커밋 
로컬저장소의 소스를 서버 저장소에 푸시
하나의 저장소가 해킹당해도 서로 백업 가능

git add > 작업폴더의 내용을 스테이징(index)으로 반영(임시 저장)
git commit > 스테이징의 내용을 로컬저장소로 최종적으로 반영(최종 저장)
git push > 로컬레파를 리모트레파로 반영
git pull > 로컬레파와 작업폴더로 리모트레파의 최신내용을 가져옴
git fetch > 로컬레파로 가져옴
